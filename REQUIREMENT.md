# REQUIREMENT.md

Vibe99 is a cross-platform terminal management app for agentic coding. This document records future-facing product requirements that are important enough to track now, but are not yet implemented.

## Planned Requirements

- Multi-window interaction: allow dragging tabs out to create a new window, and dragging tabs into another existing window.
- Agent state detection: detect working and idle states for major vibe-coding tools such as Claude Code and Codex.
- Evaluate switching the app shell from Electron to Wails if Wails can preserve the current terminal UX and cross-platform behavior with a better deployment profile.
- Experimental whole-window transparency mode: allow the Vibe99 window background to become transparent so users can see windows underneath, while keeping the focused pane fully opaque for legibility.
- Persistent user settings: preserve user-facing preferences such as font size, pane width, pane opacity, and future appearance or interaction settings across app restarts.
